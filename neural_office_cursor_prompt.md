# STAGE 2 — Walk Animation System
# ═══════════════════════════════════════════

## 🎯 Goal
Make agents stand up from their desk and walk to another agent's desk to deliver a message.
The animation must:
- Use only SVG `<animateMotion>` and `<animate>` elements (no JavaScript runtime in SVG)
- Be a continuous loop
- Have 4 distinct "scenes" — one per agent — that play sequentially
- Each scene: agent rises, walks across, pauses at target, walks back, sits

## 🚫 Constraints
- Do NOT add `<script>` tags inside the SVG (GitHub strips them)
- Do NOT use CSS `animation` with JavaScript — use declarative SVG SMIL animations only
- Do NOT touch any file except `office/generate-svg.js`
- Do NOT change any function signature that `update-office-state.js` calls

---

## 📋 EXACT TASK FOR CURSOR

Open `office/generate-svg.js`.

### Step A — Create a `drawWalkingAgent(agentId, fromPos, toPos, timing)` function

This function draws a **second copy** of the agent that is invisible by default and becomes
visible only during its animation window. It returns an SVG `<g>` group string.

`timing` is an object: `{ begin, standDuration, walkDuration, pauseDuration }`
All values are in seconds. The total scene cycle for 4 agents should be 40 seconds (10s each).

#### Scene timing layout:
```
Scene 1 (frontend → backend):   begin=0s,  walk=2s, pause=3s, return=2s, sit=3s  → total=10s
Scene 2 (backend → database):   begin=10s, same durations
Scene 3 (database → devops):    begin=20s, same durations
Scene 4 (devops → frontend):    begin=30s, same durations
LOOP: repeatCount="indefinite" on the outer animation
```

### Step B — Stand-up animation

When the scene begins, animate the walking agent's Y position upward by 20px over 0.5s.
Use `<animate attributeName="transform">` with `type="translate"`.

### Step C — Walk path animation

Use `<animateMotion>` with a straight-line `path="M fromX,fromY L toX,toY"`.
The agent should walk from their desk position to the target agent's desk position.

Walking agent should also "bob" up and down slightly:
Add a `<animateTransform>` with `type="translate"` alternating Y by ±3px every 0.2s
(simulates footstep bounce). Apply only during walk segments.

### Step D — Sit-back animation

After pause, animate return path `M toX,toY L fromX,fromY`, then animate Y back to 0
(seated position).

### Step E — Visibility gating

The walking agent group must be hidden (`visibility="hidden"`) when not active.
Use `<animate attributeName="visibility" from="hidden" to="visible" begin="Xs" dur="Ys"/>`.

### Step F — Integrate into main SVG output

In the main SVG template string at the bottom of `generate-svg.js`, add the walking agent
groups AFTER the static desk+agent groups. This ensures walkers render on top.

### Walking positions (use desk-center X, mid-body Y):
```javascript

  database: { x: 180, y: 430 },

```

### Walk routes (4 scenes):

```javascript
    { domain: 'backend',   patterns: ['backend/', 'api/', 'server/', 'routes/', '.py', 'controllers/'] },
```

```yaml
name: Update Neural Office SVG

on:
  push:
    branches: [main]
  workflow_dispatch:

permissions:

jobs:
  update-svg:

    runs-on: ubuntu-latest

      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node

      - name: Install dependencies
        run: npm ci

      - name: Run update pipeline

          git config user.name "github-actions[bot]"
          git config user.email "41898282+github-actions[bot]@users.noreply.github.com"
          git add office/base-office.svg office/office-state.json
          if git diff --staged --quiet; then
            echo "No changes to commit."
            exit 0
          fi
          git commit -m "chore(office): regenerate SVG from commit [skip ci]"
          git push
```

## 🐛 TROUBLESHOOTING GUIDE

| Symptom | Cause | Fix |
|---|---|---|
| SVG shows as broken image | SVG has `<script>` or `<foreignObject>` | Remove those tags from generate-svg.js |
| Workflow loops infinitely | Missing `[skip ci]` in bot commit | Add `[skip ci]` to the commit message in update-svg.yml |
| `git diff HEAD~1` fails in workflow | `fetch-depth` not set | Add `fetch-depth: 2` to checkout step |
| Speech lines are all generic | Groq call failing silently | Check `GROQ_API_KEY` secret is set in repo Settings → Secrets |
| Agent walks off-screen | Walk positions wrong | Adjust WALK_POSITIONS to stay within 40–820 X range |
| Animation doesn't loop | Missing `repeatCount="indefinite"` | Add to outermost animation element |
| Two agents walking at once | Scene timing overlap | Ensure each scene is exactly 10s and begin times are 0,10,20,30 |
| Text overflows bubble | No truncation | Apply `truncate(text, 34)` before drawSpeechBubble |
| SVG file >500KB | Too many particles or redundant markup | Reduce particles to 8, use CSS classes for repeated styles |

## 📌 KEY ARCHITECTURAL DECISIONS (for Cursor context)

1. **SMIL animations only** — GitHub strips `<script>` from SVGs. All animation must use
   declarative `<animate>`, `<animateTransform>`, and `<animateMotion>` elements.

2. **State is pre-baked into SVG** — The SVG is fully static once generated. There is no
   runtime data fetching. Speech lines are embedded at generation time.

3. **40-second master cycle** — Each agent gets exactly 10 seconds. The timing is hardcoded
   but values come from constants — change them in one place.

4. **Z-order convention** — Render in this order (bottom to top):
   Background → Grid → Desks → Static agents → Walking agents → Speech bubbles → UI chrome

5. **Speech lines are capped at 34 chars** — This is a hard limit for the SVG `<text>` element
   to fit inside a 200px bubble without overflow. Always truncate before rendering.
